# mcp-5e — 5E 도면 생성 MCP 서버

Claude가 **5E 프로젝트 파일(.json)을 직접 만들도록** 해 주는 도구 꾸러미다.
이미지 생성 API를 쓰지 않는다. 5E의 그림은 픽셀이 아니라 21종 객체의 JSON 스펙이고,
이 서버는 그 스펙을 좌표까지 계산해서 써 준다. 결과는 5E에서 열어 손으로 이어서 고칠 수 있다.

- **의존성 없음** — Node 18+ 만 있으면 된다. `npm install` 불필요.
- **타입 정본을 앱에서 직접 import** — `js/object-types.js`를 런타임에 읽으므로 5E에 타입이
  추가돼도 목록이 어긋나지 않는다.
- **함수 그래프는 앱과 같은 샘플러**(`js/function-graph/sampler.js`)를 호출한다 → 앱에서
  다시 그려도 곡선 모양이 어긋나지 않는다.

> **다른 교사가 자기 컴퓨터에 설치하려면** → [GUIDE_FOR_TEACHERS.md](GUIDE_FOR_TEACHERS.md)
> (터미널/개발 배경 없이 따라 할 수 있게 쓴 설치 안내서)

## 등록


## 좌표계 (제일 자주 틀리는 곳)

- 단위는 **mm** (1 world unit = 1 mm)
- 원점 (0,0) 은 아트보드 **중앙**. 좌상단이 아니다.
- +x 오른쪽, **+y 아래쪽**
- 90×60mm 아트보드 → 그릴 수 있는 범위는 `x -45~45`, `y -30~30`
  (근거: `js/svg-export.js` `exportRegion()` — 내보내기 영역이 `-w/2 … +w/2`)

## 두 가지 사용 방식

**① 열어 둔 앱에 바로 그리기 (기본)** — `add_objects` / `add_circuit` / `add_graph`에서
`path`를 **생략**하면 지금 열려 있는 5E 화면에 즉시 나타난다. 파일을 저장했다 다시 여는
왕복이 없다. 들어온 변경은 전부 **Ctrl+Z로 되돌릴 수 있다.**

동작 조건:
- 앱을 `http://localhost:…` 로 열어야 한다 (파일 더블클릭 `file://` 로는 안 된다)
- MCP 서버가 떠 있는 상태에서 앱을 **새로고침** — 앱은 켜질 때 한 번만 통로를 찾는다
- 연결되면 화면 왼쪽 아래에 `MCP 연결됨 :8579` 배지가 뜬다

**② 파일로 만들기** — `path`를 주면 `.json` 파일에 쓴다. 앱을 안 켜도 되고, 여러 페이지를
미리 만들어 두는 데 적합하다.

## 툴

| 툴 | 하는 일 |
|---|---|
| `describe_schema` | 21종 타입 요약 / 특정 타입의 기하·기본값·허용값 |
| `app_status` | 열려 있는 앱과 연결됐는지 확인 (안 되면 원인 안내) |
| `add_objects` | 객체 추가. `path` 생략 시 화면에 바로. 검증 실패 시 **하나도 넣지 않는다** |
| `add_circuit` | 사각 폐회로 — 소자 배치 + 도선 연결 + 병렬 가지 |
| `add_graph` | 좌표평면 + 함수 그래프 (수식 파서·샘플러는 앱과 동일) |
| `read_app` | 지금 화면에 뭐가 그려져 있는지 읽기 — "저거 옆에 화살표" 같은 지시를 처리할 때 |
| `remove_from_app` / `clear_app` | 화면에서 지우기 (Ctrl+Z 가능) |
| `create_project` | 빈 .json 프로젝트 생성 (아트보드 크기, 다중 페이지) |
| `list_objects` / `remove_objects` | 파일 안의 객체 조회·삭제 |
| `validate_project` | 파일이 앱에서 제대로 열릴지 검사 |

## 앱 쪽 연결부

앱에는 `js/mcp-bridge.js` 하나가 붙어 있고 `index.html`이 이걸 싣는다.

- **`localhost`/`127.0.0.1`에서 열렸을 때만** 동작한다 → GitHub Pages 배포본은 무동작
- 통로(127.0.0.1:8579~8583)가 응답하지 않으면 조용히 포기한다
- 서버 → 앱은 SSE, 앱 → 서버는 fetch POST. WebSocket을 안 쓴 이유는 의존성 없이 노드
  내장 `http`만으로 끝내기 위해서다.
- 통로는 루프백에만 바인딩되므로 외부에서 접근할 수 없다

`validate_project`가 중요한 이유: `js/project-io.js`의 로드 경로는 매우 방어적이라
필드가 틀려도 조용히 넘어가고 **그림만 이상해진다**. 파일을 만든 뒤 항상 확인한다.

## 샘플

`samples/` 의 두 파일은 이 서버가 만든 결과물이다. 5E에서 **열기**로 그대로 열어 볼 수 있다.

- `circuit-series.json` — 전지 + 저항 2개 직렬 폐회로 (소자 3개 + 도선 7개 자동 배선)
- `graph-sin.json` — 좌표평면 + `sin(x)` + 점선 직선, 그리고 사각형·볼록렌즈·화살표·각도 호

## 만들 수 있는 것 / 없는 것

**된다** — 회로도(직렬·병렬), 렌즈·거울 광선 작도, 빗면·도르래·수레 역학 배치, 자석·나침반,
함수 그래프와 좌표평면, 각도·직각 표시, 지시선 라벨, 자·각도기, 다중 페이지.

**안 된다**
- `image`(사진·래스터) — 앱에서 붙여넣기(Ctrl+V)로만 넣는다.
- 심볼 레지스트리에 없는 삽화(세포·지층·인체 등) — polyline/curve 좌표를 일일이 찍어야 해서
  품질이 급격히 떨어진다. 물리·화학은 되고 생물·지구과학 삽화는 사실상 안 된다고 보면 된다.
- 라벨 겹침 자동 회피 — 글자 폭 측정이 브라우저 캔버스에 의존해서(`js/render/graph-label.js`)
  서버 쪽에서는 못 잰다. 라벨이 겹치면 앱에서 옮긴다.
- `formula`의 `w`/`h`는 추정치가 들어간다(앱은 실측). 앱에서 한 번 만지면 실측값으로 잡힌다.

## 유지보수

```bash
node tools/mcp-5e/check-sync.mjs     # 소자·심볼 이름이 앱과 어긋났는지 검사
node tools/mcp-5e/test/smoke.mjs     # 파일 경로 전체 흐름 (기본 출력: 임시폴더)
node tools/mcp-5e/test/live.mjs      # 열어 둔 앱에 실제로 그려 보는 통합 테스트
```

`live.mjs`는 앱이 붙을 때까지 최대 40초 기다린다. 브라우저에서 5E를 localhost로 열어 두고
실행한 뒤(필요하면 새로고침) 화면을 보면 된다.

타입 목록은 앱에서 import 하므로 자동으로 맞지만, **kind/element 이름**(회로 소자·광학 심볼
등)은 렌더러 함수 이름으로만 존재해 `lib/schema.js`에 손으로 적혀 있다. 5E에 새 소자를 추가하면
`check-sync.mjs`가 알려주니 그때 같이 고친다.
