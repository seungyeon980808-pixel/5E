# 기능별 도움말 배치 제안 (v0.54 기준)

작성: 2026-07-07 · 전 기능 진입점 실사 후 작성

## 원칙: 도움말은 4개 층으로 나눈다

| 층 | 형태 | 쓰임 | 구현 비용 |
|---|---|---|---|
| ① 툴팁 | 버튼 `title` 속성 (마우스 올리면 1줄) | "이 버튼이 뭐지?" | 매우 낮음 |
| ② 상태바 힌트 | 캔버스 하단 바 오른쪽 슬롯 (자르기 안내 위치를 공용화) | 도구 사용 중 조작법 | 낮음 |
| ③ 모달 내 설명 | 해당 창 안의 설명문/? 아이콘 | 복잡한 기능의 문법·옵션 | 중간 |
| ④ 도움말 모달 | 설정 ▾ → "도움말" (또는 F1) — 전체 기능·단축키 일람 | 처음 배우기, 총람 | 중간 |

②는 이번에 자르기 안내를 하단 바로 옮기며 생긴 자리를 `#tool-hint` 공용 슬롯로 일반화하면
모든 도구가 같은 자리에 힌트를 표시할 수 있다 (화면 가림 0).

## 기능별 배치표

### 캔버스 도구 (도구 활성 중 조작법이 필요 → ①+②)
| 기능 | 현재 도움말 | 제안 |
|---|---|---|
| 선택/이동(V) | 없음 | ①툴팁 "선택·이동 (V)" + ②"드래그=이동 · Shift=스냅" |
| 자유그리기(D) | 없음 | ①"자유그리기 (D)" + ②"드래그로 그리기 · 놓으면 완성" |
| 선/사각형/원/삼각형 등 생성 도구 | 없음 | ① 각 버튼에 이름+단축키 + ②"클릭-드래그로 생성" |
| 자르기(E) | ② 있음(이번에 하단 바로 이동) | 유지 — 이 패턴이 표준 |
| 회로 기호(R·L·C·V·A 등) | 없음 | ① 기호 이름("저항", "전압계"…) 필수 — 기호만으론 초심자가 모름 |
| 스냅(Shift 드래그) | 없음 | ② 이동 중 Shift 누르면 "스냅 중: 변·접선·꼭짓점에 자석" 표시 |
| 각도 스냅(Shift+Ctrl) | 없음 | ② 위와 같은 슬롯에 병기 |

### 상단 버튼/모달 (복잡한 기능 → ③ 중심)
| 기능 | 현재 도움말 | 제안 |
|---|---|---|
| 기출 문항 검색 | 검색창 placeholder만 | ③ 모달 상단에 접이식 "검색 도움말": 압축코드 3형식·태그 AND 검색·필터 설명 |
| 이미지 객체화 | ③ 설명문 있음(양호) | 유지 + 고급(미완성) 옵션에 ? 아이콘으로 한계 설명 |
| 이미지 영역 지우기(사각형/자유) | 없음 | ① 툴팁 + ② 사용 중 힌트 |
| 비교(원본 vs 오브젝트) | 없음 | ① "원본 이미지와 그린 오브젝트를 좌우로 비교" |
| 프로젝트 저장/불러오기 | 없음 | ① 툴팁 (JSON 저장임을 명시) |
| 이미지로 내보내기 | 없음 | ③ 내보내기 대화상자 안에 형식 설명 1줄 |
| 설정 저장/불러오기 | 없음 | ① "설정을 파일로 백업/복원" |
| 라벨 편집 / 수식 폰트 | 없음 | ③ 텍스트 편집기 안에 "라벨 폰트=수식체" 안내 |

### 보이지 않는 기능 (단축키 전용 — ④ 없으면 존재를 모름)
| 기능 | 제안 |
|---|---|
| 속성 복사 Shift+C/V | ④ 도움말 모달 + 인스펙터에 ① 아이콘 버튼 추가 고려 |
| 묶기/풀기 Shift+G | ④ + 우클릭 메뉴가 있다면 그곳에 표기 |
| 객체 검색 Ctrl+F | ④ |
| 실행취소/재실행 Ctrl+Z/Y | ① (이미 버튼 있음 → title에 단축키 병기) |
| 전체화면 Alt+Enter | ① 있음(양호) |
| Space+드래그 팬 / 휠 줌 | ④ + 최초 실행 시 ② 슬롯에 1회 안내 |

## 구현 순서 제안 (효과 대비 비용)

1. **① 툴팁 일괄 추가** — index.html/템플릿에 title 속성만. 반나절, 즉시 체감.
2. **② `#tool-hint` 공용 슬롯화** — 자르기 패널을 일반화, 도구 전환 시 힌트 문자열 교체.
3. **④ 도움말 모달** — 설정 ▾에 "도움말" 항목: 기능·단축키 표 1장 (이 문서의 표 재사용).
4. **③ 기출 검색 모달 도움말** — 압축코드 문법은 지금 placeholder 한 줄로는 부족.

※ ④의 내용은 이 문서 표를 그대로 옮기면 되므로, 문서와 앱 도움말이 함께 관리된다.
