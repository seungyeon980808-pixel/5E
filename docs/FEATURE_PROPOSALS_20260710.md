## 1위. 자동 저장·크래시 복구 (로컬 스냅샷)

**무엇인가** — 작업 중인 도해를 2~3초 디바운스로 IndexedDB에 자동 저장하고, 탭 닫힘·강제 종료·정전 후 앱을 다시 열면 "이전 작업을 복구할까요?" 모달로 마지막 상태를 되살린다. 최근 스냅샷 5~10개를 롤링 보관해 시각 표시가 붙은 간이 버전 히스토리 역할도 겸한다.

**왜 중요한가** — 지금은 저장 버튼을 누르지 않으면 새로고침 한 번에 작업 전체가 증발한다. 심사 3인 중 2인이 만점을 준 유일한 항목으로, 모든 작업 시간에 상시 기여하는 투자 대비 효과 1위의 안전망이다.

**어떻게 구현하나**
- 새 파일 `js/autosave.js`. `js/project-io.js`의 내부 함수 `serialize(s)`(215행)와 `applyLoaded`(243행)를 export로 승격해 재사용.
- `main.js`에서 `initAutosave(state)` 호출 → `store.subscribe`에 디바운스를 걸어 IndexedDB(`5e-autosave` DB, 스냅샷 오브젝트 스토어)에 기록. 이미지 dataURL 때문에 localStorage 5MB로는 부족하므로 IndexedDB 필수.
- 부팅 시 미정리 스냅샷이 있으면 `ui-dialogs.js`의 `showConfirm`으로 복구 여부를 묻고 `applyLoaded` 경로로 복원. 외부 의존성 없음.

**규모**: S / **선행 조건**: 없음. 여기서 export 승격한 `serialize`/`applyLoaded`가 15위(일괄 변환)와 향후 공유 기능의 공통 토대가 된다.

## 2위. PNG 실크기(DPI) 메타데이터 삽입 — HWP 붙여넣기 크기 자동 일치

**무엇인가** — 현재 내보내기는 `canvas.toBlob()` PNG라 DPI 정보(pHYs 청크)가 없어, 한글(HWP)·워드가 96dpi로 간주해 300dpi 그림이 의도한 mm 크기의 약 3배로 붙는다. 내보내기·클립보드 복사 직후 PNG 바이너리에 pHYs 청크(픽셀/미터 = dpi ÷ 0.0254)를 삽입해 붙여넣는 순간 아트보드 mm 크기 그대로 들어가게 한다.

**왜 중요한가** — 시험지 편집의 가장 반복적인 노동인 "붙여넣고 크기 다시 맞추기"가 사라진다. 3인 전원 9점 — 한국 시험지 편집기만의 문제를 S 규모로 완결하는 차별성·완성도 대비 비용 최상 기능.

**어떻게 구현하나**
- 새 파일 `js/png-phys.js`(약 60줄): Blob→ArrayBuffer로 읽어 첫 IDAT 청크 앞에 pHYs 청크를 삽입하고 CRC32를 계산해 새 Blob을 반환하는 순수 함수.
- `js/svg-export.js`의 `exportPng()`(261행 toBlob 콜백)와 `copyPngToClipboard()`(241행)에서 blob을 이 함수로 한 번 감싸기만 하면 됨. 기존 래스터화 경로·UI 변경 없음.

**규모**: S / **선행 조건**: 없음. 완전 독립 수정이라 다른 브랜치와 병행 가능.

## 3위. 교사용/학생용 이중 내보내기 (정답 요소 숨김 학습지)

**무엇인가** — 객체에 "정답 요소" 체크를 해 두면 내보내기 시 "학생용(정답 숨김)" 옵션으로 해당 객체를 뺀 그림이 나온다. 텍스트/라벨은 빈칸 밑줄·빈 상자로 치환하는 옵션을 포함하며, 같은 파일에서 교사용(전체)·학생용을 연달아(또는 `_학생`/`_교사` 접미사 파일 2개로 한 번에) 뽑는다.

**왜 중요한가** — 학습지 제작의 실제 워크플로는 "완성 도해에서 몇 개를 지워 빈칸 문제 만들기"인데, 지금은 사본을 만들어 삭제해야 해서 원본·학생용 두 파일이 갈라지고 수정이 이중화된다. 원본 하나로 두 판본을 유지해 "학생용만 고치고 교사용은 옛 그림"인 사고를 막는다.

**어떻게 구현하나**
- 1차(객체 플래그): `js/svg-export.js`의 `isHidden(s, obj, options)`가 이미 `includeReferenceImages:false`로 옵션 기반 필터링을 하는 선례 — `options.excludeAnswers === true && obj.answerOnly` 분기 한 줄 추가로 PNG/SVG/영역/클립보드 전 경로에 일괄 적용.
- 객체 플래그는 인스펙터 공통 플래그 섹션(`js/inspector/section-protect.js`가 locked/positionLocked을 다루는 곳)에 "정답 요소" 체크박스 추가. `export-dialog.js` 모달에 "학생용" 토글과 "빈칸 상자 치환" 옵션(치환은 `buildExportSvg`에서 해당 text/formula의 bbox 자리에 rect 삽입). `project-io.js` 백필(기본 false). 캔버스에서는 정답 요소에 옅은 점선 테두리 배지(`render/annotations.js`).
- 2차(옵션): 레이어에 `role:"normal"|"answer"`를 추가(`inspector/section-layers.js` 토글)해 풀이 화살표·보조선을 레이어 단위로 대량 지정하는 경로를 병행.

**규모**: M / **선행 조건**: 없음(스키마 additive).

## 4위. 다중 아트보드 — 문항 세트 보드 (한 파일 = 시험 1회분)

**무엇인가** — 한 프로젝트 안에 보드(각자 objects/guides/layers/artboard 보유) 여러 장을 두고 캔버스 하단 탭으로 전환한다. 각 보드에 문항번호·배점 등 메타를 붙일 수 있는 그릇을 함께 만들고, "모든 보드 일괄 내보내기"로 확장한다. 시험 1회분 그림 10~20개 또는 수업 한 차시의 시각 자료 전체가 파일 하나가 된다.

**왜 중요한가** — 지금은 그림 1장=파일 1개라 중간고사 한 번에 JSON이 십수 개 생기고 스타일 통일·버전 관리가 흩어진다. 작업 단위가 실제 출제·수업 워크플로("2026 1학기 기말 과학")와 일치하게 되며, 이원목적분류표(13위)·일괄 내보내기의 토대가 된다. 유사 후보 4안이 제출될 만큼 수요가 분명하다.

**어떻게 구현하나**
- 핵심 결정: 기존 모듈 수십 곳이 참조하는 `s.objects/s.guides/s.layers/s.artboard` 구조를 절대 바꾸지 않는다. 활성 보드만 기존 자리에 두고 비활성 보드들은 `s.boards[]`(같은 4필드 묶음 + `meta:{no, points, ...}`)에 보관 — 탭 전환은 `state.update` 1회로 현재 4필드를 boards에 스왑백하고 대상 보드를 꺼내오는 방식이라 render/pick/transform/snap 전부 무수정으로 동작.
- 새 파일 `js/boards.js`: 하단 탭바 UI(추가/복제/이름변경/순서변경/삭제, 격자 컨트롤 옆). 탭 썸네일은 `templates.js`가 이미 쓰는 실제 렌더러 소축척 재사용 기법.
- undo는 v1에서 보드 전환 시 스택 초기화(전환 자체를 히스토리에 넣지 않음을 명시 — 기존 "파일 열기"와 같은 계약이라 안전).
- `project-io.js` 스키마 상향 + `migrate()`에서 구버전 파일은 보드 1개로 감싸 로드. "모든 보드 연속 내보내기"는 `export-dialog.js`의 기존 렌더 경로를 보드 루프로 감싸는 후속 확장.

**규모**: L / **선행 조건**: 스키마 대변경이므로 단독 브랜치 + 착수 전 스키마 설계 확정. 1~3위 머지 후 진행 권장.

## 5위. 정렬·분배 패널 (다중 선택 정렬 8종)

**무엇인가** — 2개 이상 선택 시 좌/중/우, 상/중/하 정렬과 가로/세로 등간격 분배 버튼 8개를 제공한다. 현재 다중 선택(마퀴+Shift클릭)은 되지만 줄 세우는 수단이 화살표 nudge뿐이다.

**왜 중요한가** — 렌즈·스크린·물체를 광축 높이에 맞추고, 회로 소자를 등간격으로 놓고, 라벨을 같은 x에 정렬하는 작업이 매 그림마다 반복된다. 모든 경쟁 편집기가 가진 기본기의 부재는 제품 완성도의 구멍이며, 눈대중 미세조정을 버튼 한 번으로 대체해 인쇄물 마감 품질을 몇 초 만에 끌어올린다.

**어떻게 구현하나**
- 새 파일 `js/align.js` + `js/inspector/section-align.js`(2개 이상 선택 시에만 노출, 기존 `section-group.js` 마운트 패턴).
- 각 선택 객체의 월드 bbox는 `js/pick.js`의 `getObjectBBox` 재사용, 이동은 `js/transform.js`의 `applyDelta`(SIZE_TYPES는 x/y 시프트, POINT_ARRAY_TYPES는 점 배열 델타 — 타입별 분기 기존재)를 호출해 델타만 계산.
- 잠긴 객체 제외 규칙과 Undo 1스텝 스냅샷은 `bulk-edit.js`와 동일 계보. 그룹 멤버는 그룹 전체를 한 단위 bbox로 취급. 스키마 변경 없음.

**규모**: S / **선행 조건**: 없음.

## 6위. 수업 발표 모드 + 단계별 공개 (스텝 리빌)

**무엇인가** — 전자칠판/프로젝터 투사용 전체화면 모드. 좌우 패널·툴바·눈금자를 숨기고 아트보드를 화면에 꽉 채운 뒤, 객체에 부여한 공개 단계 번호(`revealStep`) 순서대로 방향키/스페이스로 하나씩 누적 공개한다. PPT "나타내기" 애니메이션을 도해 편집기 안에서 구현하는 것.

**왜 중요한가** — 물체→렌즈→광선1→광선2→상(像) 순서의 단계 공개는 칠판에 순서대로 그려주는 수업 흐름을 인쇄물 품질로 재현한다. 시험용으로 만든 그림이 PPT 재작업 없이 그대로 수업 자료가 되어 "문항 편집기→수업 도구"라는 5E 고유 포지션을 만든다(3인 전원 8점).

**어떻게 구현하나**
- 새 파일 `js/present-mode.js` + `css/style.css`에 `[data-present]` 섹션, `main.js`에서 init. 전체화면은 기존 `initFullscreen` 패턴(`.app` requestFullscreen) 재사용, 진입 시 문서 루트에 `data-present` 속성을 찍어 CSS로 패널·툴바 숨김 + `viewport.js`의 `centerView`로 아트보드 맞춤.

## 7위. 실험 데이터 표 객체 + 데이터 플로터 (표 → 산점도)

**무엇인가** — 1단계: 모달에 x·y 표를 입력하거나 엑셀/시트에서 복사한 표를 붙여넣으면(TSV 파싱) 데이터 범위에 맞춰 눈금을 자동 설정한 좌표평면 위에 측정점과 꺾은선을 생성한다. 2단계: 행·열을 가진 table 객체를 신설하고, 표의 두 열을 골라 산점도로 변환하는 데이터 플롯을 얹는다.

**어떻게 구현하나**
- 1단계 `js/data-plot.js`(모달+파서+삽입): 신규 객체 타입 없이 `function-graph/insert.js`의 coordplane 자동 생성/선택 평면 삽입 경로와 `coords.js`의 수학좌표→world 매핑을 재사용해 표준 node 점 + polyline 하나를 생성 — render/pick/transform/저장이 전부 공짜(`object-types.js` 무수정). 붙여넣기는 textarea 줄 단위 split 후 탭/쉼표/공백 분리, 축 범위는 min/max 여유 10% + 1-2-5 스텝 반올림. 진입점은 좌측 "고급 기능" 버튼 + `search.js` 키워드 등록.

## 8위. 임시 판서 오버레이 (펜·형광펜·스포트라이트)

**왜 중요한가** — "이 저항 양단!" 하고 동그라미 치는 행위가 설명의 절반인데, 현행 자유 그리기는 정식 객체로 생성되어 히스토리와 저장 파일을 오염시킨다. 휘발성이라 준비 자료를 더럽히지 않고, 스포트라이트로 복잡한 도해에서 학생 시선을 모은다. state 무접촉이라 회귀 위험이 거의 없다.

**어떻게 구현하나**
- 새 파일 `js/annotate-overlay.js` 하나로 완결. `image-compare.js`에 문서화된 "순수 표시 — state/undo/export에 흔적 없음" 원칙을 따른다.

## 9위. 함수그래프 파라미터 슬라이더 + 계열 생성 (GeoGebra 슬라이더)

**무엇인가** — 함수식에 x 외 자유 문자(a, b, c, k)를 허용하고, funcgraph 선택 시 인스펙터에 파라미터별 슬라이더가 나타나 드래그하는 즉시 그래프가 다시 그려진다. 덧붙여 "계열 만들기" 버튼으로 a=1,2,3 세 값을 실선/파선/점선으로 한 번에 삽입해 비교 그래프 문항을 즉석 생성한다.

**왜 중요한가** — "기울기가 커지면?", "저항이 2배면?" 같은 조건 변화 발문이 그래프 수업의 핵심인데 지금은 식을 다시 입력해야 한다. 슬라이더는 수업 시연용, 계열 일괄 삽입은 문항 제작용 — 평가원 양식 스타일을 유지한 채 두 마리를 잡는다(3인 전원 7점).

**어떻게 구현하나**
- `function-graph/parser.js`는 순수 모듈(`compile("sin(x)") → fn`)이고 현재 변수는 x만 화이트리스트. 토크나이저 ident 처리에서 a~d, k를 만나면 params 집합에 등록하고 `fn(x, params)`로 평가하도록 시그니처만 확장(순수 함수라 회귀 위험 최소, `tests/`에 단위 테스트 추가).
- funcgraph 객체에 `params:{a:1,...}` 저장 — `project-io.js`는 일반 속성이라 자동 직렬화(+백필). `inspector/section-funcgraph.js`에 파라미터별 range 슬라이더(`inspector/widgets.js` 재사용).
- input 이벤트마다 `sampler.js`로 재샘플해 points 갱신 — 드래그 중에는 히스토리 없는 갱신, pointerup에서 1회 undo 스냅샷(`transform.js` 드래그 코얼레싱 패턴 재사용). 계열 생성은 `function-graph/insert.js`를 값 목록으로 반복 호출하며 lineStyle만 변경.

**규모**: M / **선행 조건**: 없음.

## 14위. 스타일 프리셋 라이브러리

**무엇인가** — 선택 객체의 스타일 묶음(선 굵기·선 명도·채움·점선 간격·화살표·글꼴·글씨 크기)을 이름 붙여 저장하고, 리스트 클릭 한 번으로 다른 객체(들)에 적용한다. localStorage에 영구 저장되어 프로젝트를 넘나들며 재사용된다.

**왜 중요한가** — 시험지 그림은 표준 스타일이 생명이다: 본문 도형 0.2mm 실선, 보조선 회색 점선, 라벨 9pt 같은 자기 규격을 매번 수치로 재현하는 대신 "내 시험지 표준" 클릭 한 번. 전체 통일(bulk-edit)이 "사후 교정"이라면 이것은 "그릴 때부터 규격 적용"으로, 학기 내내 수백 문항의 그림 톤이 자동 통일된다.

**어떻게 구현하나**
- 새 파일 `js/style-presets.js` — `js/personal-objects.js`와 동일 골격(localStorage 키 `5e.stylePresets`, load/save/리스트 UI). `settings.js`의 백업 키 목록(PERSONAL_KEYS)에 추가하면 기존 "설정 저장하기/불러오기"에 자동 편승해 PC 간 이동도 해결.
- 프리셋은 속성 부분집합으로 저장하고 적용 시 대상 타입에 존재하는 필드만 덮어씀(스키마 필드명이 전 타입 공통이라 안전). UI는 `js/inspector/section-style-presets.js` 접이식 섹션: "현재 선택을 프리셋으로 저장" + 프리셋 행 클릭 적용(Undo 1스텝). 커맨드 팔레트(19위)에도 등록.

**규모**: M / **선행 조건**: 없음.

## 15위. 일괄 PNG 변환기 — 프로젝트 여러 개를 한 번에 그림으로

**무엇인가** — 프로젝트 JSON 여러 개(또는 문항 세트의 모든 보드)를 선택하면 현재 열린 작업을 건드리지 않고 각각을 오프스크린 렌더해 "파일명.png"로 일괄 저장한다. File System Access API로 폴더 지정, 미지원 브라우저는 순차 다운로드 폴백. DPI·정답 요소 제외 옵션을 일괄 적용한다.

**왜 중요한가** — 시험 직전 수정이 몰릴 때(글꼴 통일, 선 굵기 일괄 변경) 그림 6~10장을 하나씩 열어 다시 내보내는 반복이 클릭 한 번으로 끝난다. 한글 문서에 다시 붙이는 순서도 파일명 정렬로 보장된다.

**어떻게 구현하나**
- 새 파일 `js/batch-export.js` 모달: `input[multiple accept=.json]`으로 파일 수집 → 각 파일을 `project-io.js` `migrate()`로 정규화(objects/layers/artboard만 갖춘 스냅샷 생성) → `svg-export.js`의 `rasterizeExportCanvas(snapshot, {dpi})`를 라이브 state 대신 스냅샷으로 호출(이미 상태 객체를 인자로 받는 시그니처라 무변경 재사용) → `showDirectoryPicker` 핸들에 순차 write, 실패 시 `a.download` 폴백.
- `migrate`/직렬화 함수의 export 공개만 추가(1위 자동저장에서 이미 승격했다면 재사용).

**규모**: M / **선행 조건**: 1위(serialize/migrate export 승격) 선행 시 구현량 감소. 2위(DPI)·3위(학생용 제외) 옵션과 결합 시 가치 상승.

## 16위. 기출 라이브러리 의미 검색 (정적 의미망 + 자연어 질의 확장)

**무엇인가** — 현재 검색은 id+제목+태그의 AND 토큰 매치라 "빗면에서 물체가 미끄러지는 그림"처럼 자연어로 치면 0건이다. ①(오프라인) tagVocab 전체에 대한 동의어·연관어 사전을 빌드 타임에 한 번 생성해 정적 JSON으로 배포. ②(선택, 후속) 매치 부족 시 질의문을 LLM으로 태그 배열로 변환해 재검색.

**왜 중요한가** — 라이브러리의 실효성은 검색 품질이 결정한다. 교사는 태그 체계를 외우지 않으므로 "자기 말"로 검색되게 하는 것이 활용률을 직접 끌어올린다. ①만으로도 체감 개선이 크고 오프라인·무비용이다.

**어떻게 구현하나**
- ①: `assets/exam-library/synonyms.json` 신설(태그별 연관어 배열, `scripts/`의 build_manifest.py 흐름에 생성 단계 추가 또는 Claude Code 세션에서 1회 산출). `exam-library.js`의 `prepareItems()`가 만드는 hay 문자열에 태그별 연관어를 이어붙이면 기존 AND 검색이 그대로 동작 — 수정 지점이 함수 1개.
- ②: `js/ai/search-expand.js` 신설, 검색 결과 0~2건일 때만 "AI로 다시 검색" 버튼 노출. manifest는 첫 오픈 시 1회 로드하는 기존 규약 유지.

**규모**: S(①) / **선행 조건**: ①은 없음. ②는 BYOK AI 코어(이번 20선 밖)가 선행이므로 ①만 우선 출시.

## 17위. 기출 출처 자동 표기 — 삽입 이력·변형 표시

**무엇인가** — 기출 라이브러리에서 문항을 삽입/객체 변환할 때 연도·월·과목·문항번호 메타를 결과 객체의 source 필드에 자동 기록한다. 인스펙터의 [출처 라벨 삽입] 버튼으로 "[2026학년도 수능 물리Ⅰ 11번 변형]" 형식 텍스트 객체를 한 번에 생성하고, 프로젝트 JSON에도 사용 기출 목록이 남는다.

**왜 중요한가** — 기출 활용·변형 문항의 출처 표기는 출제 관행이자 검토 지적 단골인데, 삽입 시점에 자동 기록되므로 "이 그림 어느 기출이었지?"를 찾는 시간이 사라진다. 기출 라이브러리라는 5E 고유 자산에 S 규모로 확실한 "출제 도구"다움을 더한다.

**어떻게 구현하나**
- `exam-library.js`의 `insertItem()`(331행)·`objectifyItem()`(347행)에서 item의 `{year, month, subjectLabel, no}`를 넘겨 image 객체에 source 필드로 저장(`insertImageFromSrc`에 옵션 인자 추가).
- `inspector/section-image.js`에 출처 표시 + 라벨 생성 버튼 — 라벨은 기존 text 객체 생성 경로 재사용. `project-io.js`는 필드 하나 보존이면 충분(additive).

**규모**: S / **선행 조건**: 없음.

## 18위. 같은 속성 선택 (Select Same) + 선택 세트

**무엇인가** — 기준 객체 하나를 잡고 "같은 종류 / 같은 선 스타일 / 같은 점선 / 같은 채움"인 객체를 캔버스 전체에서 한 번에 선택한다. 덤으로 현재 선택을 이름 붙여 저장했다가 복원하는 선택 세트(세션 한정)도 제공.

**왜 중요한가** — 기존 bulk-edit은 대상이 "선택한 것" 아니면 "전체" 둘뿐이라 중간이 없다 — "보조 점선만 전부 회색으로"를 하려면 일일이 Shift클릭해야 한다. 이 빈칸을 메워 일괄 편집·정렬(5위)·스타일 프리셋(14위)의 대상 지정 정밀도를 끌어올리는 저비용 보조 장비다.

**어떻게 구현하나**
- 새 파일 `js/select-same.js`(소형). `selectedIds[0]`를 기준으로 comparator 4종(type / strokeLevel+strokeWidth / dashLength+dashGap / fillLevel+fillNone)을 `state.objects`에 순회 적용, 가시 레이어 필터는 `snap.js`의 `isSnapTargetEligible`과 같은 규칙 재사용, 결과로 `s.selectedIds` 교체(상태 변경만이라 Undo 불필요).
- 선택 세트는 모듈 레벨 Map(이름→ids, 복원 시 존재 id만). UI 비용 최소화: 커맨드 팔레트(19위) 명령 4개로 노출하면 신규 패널 없이 완결, 팔레트 도입 전이면 인스펙터 선택 섹션 드롭다운 버튼 1개.

**규모**: S / **선행 조건**: 없음(19위 팔레트가 있으면 UI 비용 0).

## 19위. 커맨드 팔레트 (Ctrl+K 통합 실행기)

**무엇인가** — Ctrl+K로 여는 타이핑 실행기. 기존 Ctrl+F 오브젝트 검색에 더해 "명령"(그룹 묶기, 내보내기, 격자 토글, 정렬, 설정 열기 등)을 같은 창에서 검색·실행한다. 각 항목 옆에 단축키를 표기해 단축키 학습 UI를 겸한다.

**왜 중요한가** — 기능이 이미 20종+ 도구·고급기능·메뉴 3곳에 흩어져 있고, 좌측 아이콘 그리드에서 도구 찾기가 어렵다는 불편이 실사용에서 확인됐다. 이름만 알면 즉시 실행되므로 기능이 늘수록 커지는 탐색 비용을 상수로 고정한다. 14·18위 등 신기능의 저비용 노출 채널이기도 하다.

**어떻게 구현하나**
- 새 파일 `js/command-palette.js`. `js/search.js`의 modal-overlay+키보드 내비 구조를 복제하고 데이터 소스만 이원화: (a) 오브젝트는 기존 TEMPLATES/listPersonalItems에 위임, (b) 명령은 새 레지스트리 `COMMANDS=[{id, label, keywords, shortcutLabel, run}]` — run은 기존 export 함수 호출(undo/redo, 그룹, export-dialog 열기 등)이라 신규 로직 없음.
- 각 모듈이 `registerCommand()`로 등록하는 방식이면 결합도 없이 확장. 최근 실행 항목은 localStorage 저장해 상단 노출. 기존 Ctrl+F는 그대로 두고 팔레트에서 통합 검색.

**규모**: M / **선행 조건**: 없음.

## 20위. 반복 복제·배열 스탬프 (Ctrl+D + 배열 생성기)

**무엇인가** — ①Ctrl+D 즉시 복제(직전 이동 델타를 기억해 연타 시 같은 간격으로 계속 찍히는 "변형 반복"), ②배열 모달(행×열 격자 배열·간격 mm 지정·중심점 기준 원형 배열). 현재 복제 수단은 Ctrl+C/V(커서 위치 붙여넣기)뿐이다.

**왜 중요한가** — 과학 도해는 반복 패턴 덩어리다: 파동 마루·골, 이중슬릿, 등전위선, 직렬 저항, 원형 배열은 시계 눈금·나침반 배치에 바로 쓰인다. "하나 그리고 → 간격 잡고 → N개 찍기"가 한 동작이 되면 반복 도해 제작이 수 분에서 수 초로 준다.

**어떻게 구현하나**
- 새 파일 `js/duplicate.js`. 단발 복제는 `js/transform.js`의 `instantiateObjectsAt`(id·groupId 재부여+Undo 1스텝 완비)를 그대로 호출하고 target만 "선택 bbox 중심+기억 델타"로 계산. 직전 이동 델타는 transform.js body-move 커밋부에 export 한 줄 추가로 노출.
- 배열 모달은 `bulk-edit.js`의 모달 골격을 복제해 행/열/간격/원형(개수·중심각) 입력 → 루프로 instantiate. 원형 배열의 점 회전은 bulk-edit.js에 이미 있는 `rotatePt` 재사용.
- DESIGN 4-2의 "Ctrl+D 없음"은 되돌리기 싼 결정이므로 단축키 재지정 가능함을 착수 프롬프트에 명기.

**규모**: M / **선행 조건**: 없음.

## 추천 착수 순서: 다음 3개 브랜치

세 건 모두 서로 파일 충돌이 거의 없어 워크트리로 병행 가능하다.

1. **`feat/autosave`** — 1위 자동 저장·크래시 복구 (S)
   - 최고 점수(28)이면서 규모 S. `project-io.js`의 `serialize`/`applyLoaded` export 승격이 부산물로 남아 15위(일괄 변환)와 향후 공유 계열 기능의 공통 토대가 된다. 데이터 유실 리스크 제거가 다른 어떤 기능보다 먼저다.
2. **`feat/png-dpi`** — 2위 PNG 실크기(DPI) pHYs 삽입 (S)
   - `js/png-phys.js` 신설 + `svg-export.js` 2곳 래핑뿐인 완전 독립 수정. 매 내보내기마다 무조건 체감되는 개선을 가장 싸게 얻는다.
3. **`feat/dual-export`** — 3위 교사용/학생용 이중 내보내기 (M)
   - 스키마 additive(객체 `answerOnly` 플래그)라 안전하고, 내보내기 경로 중심이라 위 두 브랜치와 충돌이 최소다. 매주 반복되는 학습지·정답지 이중 관리 문제를 바로 해소한다.

**그 다음**: 4위 다중 아트보드는 스키마 대변경(L)이므로 위 3개가 integration-hub에 머지된 뒤 **`feat/multi-board`** 단독 브랜치로 스키마 설계 확정 → 구현 순서를 권장한다. 5위 정렬(S)·8위 판서 오버레이(S)는 그 사이 틈새에 끼워 넣기 좋은 소형 브랜치 후보다.
