# 이미지 객체화 — 고급 선·도형 승격 확정 명세

2026-07-06 · 브랜치 `feat/objectify-line-extraction` · 명세 인터뷰(Q1~Q17)로 확정.
이 문서가 구현의 단일 기준이다. 대화 기록과 충돌하면 이 문서가 우선.

---

## 0. 헌법 (모든 판단에서 최우선)

1. **한 시각 요소 = 한 객체.** 선은 stroke(획) 객체, 테두리 있는 도형은 stroke+fill 한 객체.
   **"다른 색 도형 2개를 겹쳐서" 표현하는 것 금지** (예: 검은 테두리 폴리곤 + 흰 구멍 폴리곤).
2. **겉보기 동일.** 삽입 결과는 원본 이미지와 육안으로 같아야 한다. 색·굵기는 실측 반영
   (굵기 px→mm 환산, 회색조 유지). 도구 기본값으로 정규화하지 않는다.
3. **이 기능 전체는 '고급 옵션'.** 토글 기본 **꺼짐**. 꺼짐 = 현행과 100% 동일(회귀 0).
   판정이 애매하면 조용히 현행 결과로 폴백. **새 기능이 현행보다 결과 이미지 퀄리티를
   저하시키는 것은 어떤 경우에도 실패다.**

## 2. 판정 사다리 (컴포넌트 처리 순서)

기존 fitComponentEllipse의 "판정 성공 → 상위 객체, 실패 → 아래 단으로 폴백" 철학을 확장.
**고급 토글이 꺼져 있으면 (신규) 표시된 단은 전부 건너뛴다.**

```
컴포넌트
 ├─ ① 텍스트인가? (기존 isText)
 │     → [옵션] 이미지 크롭 유지(기본) / formula placeholder로 대체(Q5) / 남기기 / 지우기
 ├─ ② 원/링인가? (기존 fitComponentEllipse) → ellipse
 ├─ ③ 사각 링인가? (신규 fitComponentRect)  → 회전 rect (stroke+fill 한 객체)
 ├─ ④ 닫힌 균일두께 띠인가? (신규 strokedRegion) → stroke+fill closed polyline
 │     · 띠의 구멍 안을 밝은 레벨 컴포넌트가 채우면 그 회색을 fill로 흡수(병합)
 ├─ ⑤ 가는 획 망인가? (신규 extractStrokes)
 │     · 세선화(중심선) → 분기점·급꺾임에서 토막 → 경로별 판정:
 │       곧다 → line / 꺾은선 폴리라인 → polyline(열린) / 휘었다 → curve(열린)
 │     · 굵기·회색조 실측 → strokeWidth·strokeLevel
 │     · 획으로 설명 안 되는 잔여 잉크(화살촉 등) → 현행 방식 조각(폴백 폴리곤)으로 함께 방출
 └─ 폴백: 현행 closed polyline/curve (기존 동작 그대로)
```

**부분 방출 + 잔여 조각**이 핵심 설계다: 화살촉 달린 광선처럼 "가는 획 + 두꺼운 덩어리"가
한 컴포넌트일 때, 가는 경로만 선으로 승격하고 나머지는 현행 조각으로 남긴다(Q7).

### 판정 게이트 시작값 (테스트로 조정, 최종값은 여기 갱신)
- 획: 평균 굵기 ≤ 8px, 굵기 변동계수 ≤ 0.35, 길이 ≥ 3×굵기
- 컴포넌트 승격: 획들이 잉크의 ≥ 60%를 설명해야(미달 시 컴포넌트 전체 현행 폴백)
- 직선 판정: 현(chord) 이탈 ≤ max(1.5px, 현 길이의 4%) / 꺾임 분할 각 ≥ 35°
- rect: RDP 후 꼭짓점 4개, 각 90°±10°, 면적비 ≥ 0.92, 띠 두께 CV ≤ 0.30
- 윤곽+채움 병합: 구멍 면적의 ≥ 80%를 단일 밝은 컴포넌트가 채울 때만

## 3. 산출 객체 매핑

| 래스터 요소 | 산출 객체 | 필드 |
|---|---|---|
| 곧은 획 | `line` | p1/p2, strokeWidth=실측px×scale, strokeLevel=실측 회색, arrowHead:"none", dash 0 |
| 꺾은 획(토막 후에도 여러 마디) | `polyline` 열린 | points, closed:false, fillNone:true, 실측 stroke |
| 휜 획 | `curve` 열린 | points, closed:false, fillNone:true, 실측 stroke |
| 사각 링(+내부 채움) | `rect` | x/y/w/h/rotation, strokeLevel/strokeWidth=띠 실측, fillLevel=내부 실측(비면 흰색) |
| 비(非)사각 균일 띠(+내부 채움) | `polyline` closed | stroke+fill 한 객체 (겉보기 오차 ≤ 2px 허용) |
| 텍스트(대체 선택 시) | `formula` | source=순번 placeholder(A,B,C…), 기존 text 배치 로직의 위치·크기 계승 |
| 화살촉·직각표시·각도호·dash 조각 | 현행 조각 | 폴백 폴리곤 (인식 안 함) |
| 그 외 전부 | 현행 그대로 | closed polyline/curve, 이미지 크롭 등 |

## 4. UI

- 전처리 모달 체크박스 1개: **"[고급] 선·도형 승격 (획→선 객체, 사각→상자, 테두리+채움 통합)"**
  — 기본 꺼짐. `currentOptions()`에 `advancedShapes:boolean`으로 전달.
- 켜면 **판정 미리보기**: 분석 후 미리보기 캔버스에서 컴포넌트를 판정별 색으로 살짝 틴트
  (선=파랑 #0969da, rect=초록, 균일띠=청록, 원=보라, 텍스트=주황, 폴백=현행 표시).
  범례에 같은 색 설명 추가. 기존 클릭=제외 UX 유지.
- 슬라이더 추가 없음. 임계값은 내부 상수.

## 5. 안전·폴백 원칙

- 모든 신규 판정 함수는 실패 시 `null` 반환 → 호출부는 다음 단으로. 예외를 밖으로 던지지 않는다.
- 신규 코드는 `advancedShapes` 게이트 안에서만 실행 (꺼짐 = 코드 경로 자체가 현행과 동일).
- 유일한 무조건 변경: textMode='replace'가 text 대신 **formula**를 만드는 것 (Q5 명시 합의).
- 검증 필요 항목: formula 객체가 한글('공기')을 수용 가능한지 — 렌더가 흉하면 백로그에 기록만
  (placeholder는 ASCII 순번이므로 v1 차단 요소 아님).

## 6. 범위 제외 (후속 백로그)

점선 병합(dash 군집→dashLength line 1개) · 화살촉→arrowHead 승격 · 직각/각도호 심볼 인식 ·
자동 그룹 · OCR · 교차 관통 재구성(토막의 반대) · 슬라이더 노출

## 7. 수용 기준 (DoD)

픽스처 = ①프리즘 광선도 ②도르래 경사면 (사용자 제공 시험 이미지).
1. 픽스처에서 선(직선·곡선)이 stroke 객체로, 상자·삼각형이 stroke+fill 한 객체로 나온다 — 겹친 두 도형 없음
2. 텍스트 "대체" 선택 시 formula placeholder가 들어간다
3. 토글 꺼짐 = 현행과 100% 동일 (회귀 0)
4. 겉보기: 삽입 결과가 원본과 육안으로 같다 — **최종 판정은 사용자**
5. Node 테스트 전부 통과 + 브라우저 콘솔 에러 0
6. 판정 미리보기 색 표시가 동작한다
