# QA 픽스처 사용 안내

이 폴더는 `docs/QA_VISUAL_CHECKLIST_20260701.md`와 함께 쓰는 **수동 시각 QA용 고정 입력 세트**다.
앱 동작을 바꾸지 않으며, 동일한 그림을 매번 똑같이 불러와 회귀를 눈으로 검증하기 위한 자료다.

## 파일 목록

- `label_text_export_fixture.json` — 라벨러·각도호·글꼴·라벨·선택영역 export를 한 번에 검증하는 프로젝트 픽스처.
  저장 스키마 `0.15` (`js/project-io.js`)에 맞춰 작성됨.
- `all_types_v0.15.json` — **전 객체 타입 18종(22개 객체) 호환성 픽스처** (v0.40.0, 2026-07-03).
  실행 중인 앱에서 실제 생성 경로로 만든 객체를 저장 포맷 그대로 추출한 것.
  스키마를 바꾸는 커밋 전후로 이 파일을 불러와 22개 객체가 전부 그려지는지 확인한다.
  타입 목록은 `docs/OBJECT_SCHEMA.md` §1과 동일.
- `label_text_export_expected.md` — 픽스처를 불러왔을 때 화면이 어떻게 보여야 하는지(객체별 기대 모습).
- (대체용) `label_text_export_fixture_MANUAL.md` — 불러오기 흐름이 막힐 때 앱에서 직접 만드는 수동 단계.
  현재는 JSON 픽스처가 정상 작성되어 있어 **생성하지 않았다.** 필요 시 expected 문서의 객체 목록으로 재현하면 된다.

## 1. 로컬 서버 실행

이 앱은 ES 모듈을 사용하므로 `file://`로 열면 안 되고 반드시 HTTP 서버를 경유해야 한다.

프로젝트 루트(`51_phy_draw_web`)에서:

```bash
python -m http.server 8000
```

## 2. 앱 열기

브라우저에서:

```
http://localhost:8000
```

> 권장 브라우저: **Chrome 또는 Edge**.
> (export 저장 위치를 고르는 File System Access API picker가 Chromium 계열에서만 동작하며,
>  미지원 브라우저에서는 자동으로 일반 다운로드로 폴백된다.)

## 3. 픽스처 불러오기 (앱이 프로젝트 import를 지원하는 경우)

1. 상단 메뉴의 **파일 → 열기**(프로젝트 열기) 버튼을 누른다.
2. 파일 선택 창에서 `docs/qa-fixtures/label_text_export_fixture.json`을 고른다.
3. 캔버스에 픽스처 객체들이 나타나면 성공이다. (불러오면 실행취소 히스토리·선택은 초기화된다.)

불러오기 동작은 `js/project-io.js`의 `openProject` → `migrate` → `applyLoaded` 경로를 그대로 탄다.
형식 오류 시 앱은 경고만 띄우고 **현재 작업 상태를 훼손하지 않는다.**

## 4. 불러오기가 안 될 때 (수동 재현)

import가 막히거나 스키마가 달라 경고가 뜨면, `label_text_export_expected.md`의
**객체 목록**을 보고 같은 종류·텍스트·대략적 위치로 직접 그려 넣는다. 정확한 좌표보다
**종류·텍스트·글꼴 정책·배치(상하좌우 분산)**가 맞는지가 검증 핵심이다.

## 5. export 확인 항목

체크리스트 D·E 섹션과 함께 확인한다.

- **전체 PNG export**: 흰 배경, 아트보드 영역 전체. 글자/라벨이 캔버스와 동일한지.
- **전체 SVG export**: 투명 배경, 벡터, 실제 mm 크기. 텍스트가 `<text>`로 살아 있는지.
- **선택영역 export**: 드래그한 월드 영역만 정확히 잘리는지 (기본 줌 / 줌 인 / 줌 아웃 / pan 후).
- **공통**: 가이드·선택핸들·마퀴·스냅 미리보기·그리드는 export에 포함되면 안 된다.
- **글꼴 주의**: 글꼴은 export에 임베드되지 않는다. 내보낸 PDF/문서에서 글꼴이 달라 보이면
  대상 기기의 글꼴 설치 문제이지 앱 버그가 아닐 수 있다(체크리스트 C6 참고).
