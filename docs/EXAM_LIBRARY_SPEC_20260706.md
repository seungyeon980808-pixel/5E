# 기출 이미지 라이브러리 스펙 (EXAM_LIBRARY_SPEC_20260706)

브랜치: `feat/exam-image-import`. 서버·API 없이 정적 파일만으로 동작하는 기출 문항 이미지 검색·삽입 기능의 데이터 규약.

## 검색 방식 3가지 (전부 AND 결합)

1. **코드/문항번호 입력** — 검색창에 `2026 수능 1`, `p1 2025 11` 등. 토큰이 모두 id·제목·태그에 포함되면 매치.
2. **드롭다운 선택** — 과목 · 파트(역학/전자기학/광학/현대물리학) · 년도. 태그가 없어도 과목·년도는 항상 동작.
   **과목 드롭다운은 하드코딩이 아니라 manifest에 실제로 존재하는 과목만 자동으로 채워진다**
   (물리→화학→생명→지구→통합 순 정렬, `js/exam-library.js`의 `SUBJECT_ORDER`). 새 과목을 넣어도
   코드 수정 없이 그대로 옵션에 나타난다.
3. **태그 칩** — 파트별로 묶인 개념 칩을 눌러 필터.

세 방식은 서로 배타적이지 않고 함께 좁혀진다(예: 파트=역학 + 년도=2026 + 검색 "8").

## 확정된 결정

1. **이미지는 로컬 전용** — 평가원·교육청 기출 이미지를 공개 레포에 올리면 공개 재배포가 되므로 `assets/exam-library/images/`와 `manifest.json`은 `.gitignore` 처리. 코드·어휘집·태그 시트만 커밋한다.
2. **태깅은 엑셀 + 생성 스크립트** — `tags.xlsx`에서 태깅하고 `build_manifest.py`가 매니페스트를 생성.
3. **파일명은 식별자 전용** — 한글 라벨은 매니페스트가 자동 생성 (파일명에 라벨을 붙여도 무시되므로 허용).

## 폴더 구조

```
assets/exam-library/
├── .gitignore        images/*, manifest.json 제외 (커밋 방지)
├── tag-vocab.json    태그 통제 어휘집 (커밋됨 — 새 태그는 여기 먼저 추가)
├── tags.xlsx         문항별 태그 시트 (커밋됨 — id | tags 두 열)
├── manifest.json     생성 산출물 (스크립트가 만듦, 커밋 안 됨)
└── images/           기출 PNG (로컬 전용, 커밋 안 됨)
scripts/build_manifest.py   매니페스트 생성 스크립트
docs/qa-fixtures/exam-library/   QA용 자작 도식 6장 (저작권 무관, 커밋됨)
```

## 파일명 규칙

`<과목코드>_<학년도 4자리>_<시험월 2자리>_<문항번호>[_<서브번호>].png`

- 예: `p1_2026_11_01.png` → **2026학년도 수능 물리1 1번**
- 예: `p1_2026_11_02_1.png`, `p1_2026_11_02_2.png` → **한 문항에 그림이 여러 장**일 때 서브번호(`_1`, `_2`, ...)로
  구분. id도 서브번호까지 포함해 서로 달라지므로(`p1_2026_11_02_1` / `_2`) 각각 별개 카드로 검색·삽입된다.
  제목에는 자동으로 `(1/2)`처럼 총 개수가 붙는다. 그림이 1장뿐이면 서브번호를 안 붙여도 되고, 붙여도(`_1`) 무방.
- 과목코드: `p1`=물리1, `p2`=물리2, `c1`/`c2`=화학, `b1`/`b2`=생명, `e1`/`e2`=지구, `i1`=통합과학
- 시험월 → 시험 종류: `11`=수능, `06`/`09`=모평, 그 외(`03`,`04`,`07`,`10`)=학평
- **연도는 학년도 기준**: 수능·모평은 시행 다음 해가 학년도 (2025년 11월 시행 = 2026학년도). 학평은 시행 연도 = 학년도.
- 프리픽스 뒤 내용은 무시되므로 `p1_2025_03_02 [2025학년도 3월 학평 물리1 2번].png`처럼 사람이 읽을 라벨을 붙여도 된다. 단 같은 문항·서브번호의 파일이 두 개면(완전히 같은 id) id 중복 경고 후 뒤의 것을 건너뜀.

## 파트(단원) 분류

파트 = 4개 고정: **역학 / 전자기학 / 광학 / 현대물리학** (`tag-vocab.json`의 `parts`).
**현재 이 어휘집은 물리 전용**이라 화학1·생명1·지구1 등 다른 과목 문항은 태그를 달아도 파트가
비어 있다(검색·삽입 자체는 정상 동작 — 파트 드롭다운·칩만 해당 없음). 다른 과목의 파트 분류가
필요해지면 `tag-vocab.json`에 그 과목의 카테고리를 추가하면 된다(별도 과제).
각 문항의 파트는 두 경로로 정해진다:

- **자동 도출(기본)** — 태그가 어느 파트에 속하는지는 `tag-vocab.json`의 카테고리(= 파트)로 정해져 있어, 문항에 태그만 달면 파트가 따라온다. 한 문항이 여러 파트 태그를 가지면 파트도 여러 개.
- **명시 지정(선택)** — `tags.xlsx`의 C열(part)에 파트명을 적으면 태그와 무관하게 그 값을 쓴다. 개념 태그를 아직 안 달았지만 파트만 빠르게 분류하고 싶을 때 유용.

태그를 다른 파트로 옮기려면 `tag-vocab.json`에서 해당 태그를 옮긴 뒤 스크립트 재실행(문항 데이터는 안 건드림).

## 태깅 워크플로

1. `images/`에 PNG를 넣는다 (파일명 규칙 준수).
2. `tags.xlsx`에 행 추가: **A열 = id** (`p1_2026_11_01`), **B열 = 태그** (쉼표 구분), **C열 = part**(선택).
   - 태그는 **`tag-vocab.json`에 있는 것만** 사용 (띄어쓰기 없이). 새 개념이 필요하면 어휘집에 먼저 추가.
   - C열은 비워도 됨 → 파트는 태그에서 자동 도출.
3. `python scripts/build_manifest.py` 실행.
   - 파일명 규칙 위반, id 중복, 이미지 없는 태그 행, 어휘집에 없는 태그를 전부 경고로 보고.
4. 앱 새로고침 → 검색 모달이 새 manifest를 읽음.

태그가 없어도 문항번호·과목·년도 검색은 동작한다 (제목·과목·년도가 파일명에서 자동 생성되므로). 태깅은 뒤로 미룰 수 있는 작업.

## manifest.json 스키마 (`exam-library-v1`)

```json
{
  "version": "exam-library-v1",
  "generated": "2026-07-06 14:32",
  "count": 6,
  "parts": ["역학", "전자기학", "광학", "현대물리학"],
  "years": [2026, 2025],
  "tagVocab": [ { "name": "역학", "tags": ["빗면", "..."] } ],
  "items": [
    { "id": "p1_2026_11_01", "file": "p1_2026_11_01.png",
      "subject": "p1", "subjectLabel": "물리1",
      "year": 2026, "month": 11, "exam": "수능", "no": 1,
      "title": "2026학년도 수능 물리1 1번",
      "tags": ["빗면", "마찰력", "뉴턴운동법칙"],
      "parts": ["역학"] },
    { "id": "p1_2026_11_02_1", "file": "p1_2026_11_02_1.png",
      "subject": "p1", "subjectLabel": "물리1",
      "year": 2026, "month": 11, "exam": "수능", "no": 2,
      "title": "2026학년도 수능 물리1 2번 (1/2)",
      "tags": [], "parts": [] }
  ]
}
```

`parts`·`years`·`tagVocab`을 매니페스트에 포함하는 이유: 앱이 드롭다운 옵션과 태그 칩을 그릴 때 fetch 1회로 끝내기 위해 (어휘집·파일 목록을 따로 안 읽음). `tagVocab`의 카테고리 `name`이 곧 파트다.

## 성능 규약 (앱 쪽 구현이 지켜야 할 것)

- 앱 시작 시 라이브러리 관련 로드 **0** — manifest fetch는 검색 모달 첫 오픈 시 1회.
- 이미지는 결과 그리드에서 `loading="lazy"`로 화면에 보이는 것만 로드.
- 검색은 클라이언트 선형 스캔 (1600건 < 1ms, 라이브러리·인덱스 불필요).

## QA 픽스처

`docs/qa-fixtures/exam-library/`의 자작 도식 6장 (빗면, 용수철, 포물선, 회로, 파동, 자유낙하). 테스트 시 `assets/exam-library/images/`로 복사해 사용. 라벨 붙은 파일명 1장 포함 (규칙 관대함 검증용).
