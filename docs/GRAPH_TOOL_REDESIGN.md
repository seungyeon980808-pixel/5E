## 0. 왜 개편하나 (진짜 문제)

지금 그래프 도구는 **"만들고 나서 인스펙터에서 상세설정"** 하는 구조라, 만들 때와 고칠 때 설정이 두 군데(모달 + 인스펙터)로 갈려 있다. 이 때문에:
- 반복해서 "화면에서 봐야 아는" 미세조정이 오갔고(눈금·격자·라벨·굵기), 매번 어긋났다.
- 함수 그래프 상세설정(선 종류·선 모양·표시점·수선·구간 화살표)이 인스펙터에 흩어져 있어, **완성본을 한 번에** 만들지 못한다.

**원하는 결과**: 모달 안에서 **좌표 → 함수 → 요소까지 미리 다 세팅**하고 "만들기" 한 번에 완성본이 나온다. 만든 뒤엔 더블클릭으로 같은 모달을 다시 열어 고친다. 인스펙터는 비운다. 기존 "함수 입력(F)" 도구의 익숙한 편집 경험을 그대로 가져온다.

## 1. 확정 사항 (대화로 못 박음)

| # | 확정 |
|---|---|
| 1 | **눈금 표시선(tick marks) 색 = 회색.** 축선·화살표·글자(축이름/눈금숫자)는 검정 유지. 배경 격자는 현행(옅은 회색 점선) 유지. |
| 2 | 그래프 모달을 **2탭**으로: **[좌표]** 탭 + **[함수]** 탭. |
| 3 | **함수 탭 하나에 함수식과 직선·꺾은선을 둘 다** 담는다(칩으로 `＋함수식`·`＋직선·꺾은선` 추가). 기존 함수 도구에 '직선·꺾은선' 종류만 더한 형태. |
| 4 | **표시점(●)·수선의 발·구간 화살표(그래프 요소)를 함수 탭에서 함수별로** 미리 세팅한다(만들기 전에 완성). |
| 5 | 만든 뒤 **인스펙터는 비운다**. 재편집 = 그래프를 **더블클릭 → 같은 모달 다시 열기**. |
| 6 | **F 단축키 = 이 통합 탭 모달**을 연다(함수 탭 포커스). 그래프 입구가 하나로 통합. |

## 2. 새 모달 레이아웃

```
┌────────────────────────────────────────────────────────────┐
│  그래프 만들기                                         [×]  │
│  ┌───────────┬───────────┐                                 │
│  │ ① 좌표     │ ② 함수     │   ← 탭 (클릭 전환)              │
│  ├───────────┴───────────┴──────────────┬──────────────────┤
│  │  (선택된 탭의 설정 패널)               │                  │
│  │                                       │   미리보기        │
│  │  좌표 탭: 형태/칸수/축이름/원점/       │   (항상 표시,     │
│  │           격자/눈금/눈금라벨/글씨/묶기 │    양 탭 공통)    │
│  │                                       │                  │
│  │  함수 탭: 계열 칩 + 선택계열 편집기    │   좌표·함수·요소  │
│  │           + 그래프 요소               │   전부 실시간     │
│  │                                       │                  │
│  └───────────────────────────────────────┴──────────────────┤
│                                   [취소]   [만들기 / 적용]   │
└────────────────────────────────────────────────────────────┘
```

- **미리보기는 오른쪽에 항상** 뜬다(탭과 무관). 좌표를 바꾸든 함수를 넣든 즉시 반영 → "미리 다 세팅"의 핵심.
- 탭은 **설정 패널만** 전환. 미리보기·하단 버튼은 고정.
- 편집 모드로 열면 제목 "그래프 편집", 버튼 "적용". 마지막에 보던 탭 기억(선택).

> ⚠️ 가정 A: 미리보기는 두 탭이 공유하는 오른쪽 고정 영역. (탭마다 따로 미리보기 X) — 아니라면 알려주세요.

---

## 3. ① 좌표 탭 (현행 "① 좌표 틀" 이동)

지금 모달의 좌표 설정을 **그대로 이 탭으로** 옮긴다(신규 개발 없음, 배치만):
- 형태(ㄴ자/ㅏ자/십자) · 가로·세로 칸 수(−/＋) · 가로·세로축 이름(수식·한글) · 원점 토글(0/O) · 격자·눈금·원점 표시 · 눈금 라벨(없음/숫자/직접) · 글씨 크기(%) · **좌표와 함수를 하나로 묶기(함께 이동)**.

---

## 4. ② 함수 탭 (기존 함수 도구 + 확장)

기존 `함수 입력` 모달([modal.js](../js/function-graph/modal.js))의 UX를 재사용하되, **계열 종류·선 모양·그래프 요소**를 더한다.

### 4-1. 계열 목록(칩)
- 상단에 계열 칩(선택/삭제). 추가 버튼 2개: **`＋함수식`**, **`＋직선·꺾은선`**.
- 함수식 계열 → `y=` 수식 편집. 직선·꺾은선 계열 → 미리보기 클릭으로 점 찍기(반 칸 스냅) 또는 좌표 텍스트 입력. (현행 그래프 모달의 두 방식 유지)

### 4-2. 선택 계열 편집기 (함수 하나를 여기서 완결)
| 항목 | 함수식 | 직선·꺾은선 |
|---|---|---|
| 수식 `y=` + 도우미 버튼(sin/cos/√/π…) | ○ | — |
| 정의역 + 미리보기 세로 핸들 드래그 | ○ | — |
| 점 입력(클릭/좌표 텍스트, Enter·우클릭 완료) | — | ○ |
| 선 굵기 | ○ | ○ |
| 선 종류(실선/점선/파선) | ○ | ○ |
| **선 모양(직선/곡선)** | ○ | ○ |
| 끝 라벨(수식 가능) | ○ | ○ |

### 4-3. 그래프 요소 (함수별, 만들기 전에 미리 얹기 — 확정 4)
선택 계열 편집기 안에 접이식/구획으로:
- **표시점 ●** : 정의역 x 입력 → 추가(여러 개 목록)
- **수선의 발** : 정의역 x 입력 → 추가(x·y축 점선 2개)
- **구간 화살표** : x₁~x₂ + **방향 4옵션**(끝→/끝←/중간→/중간←) → 추가

미리보기에 표시점·수선·화살표까지 **실시간으로** 그려서, "만들기" 전에 완성 모습을 본다.

**입력 방식 = 미리보기 클릭 + x값 숫자 입력 둘 다**(확정): 미리보기 함수 위를 **클릭**해 표시점/수선을 바로 찍고, 정확한 위치는 **x값 숫자**로 미세조정. 계열별 목록으로 여러 개 쌓이며 미리보기 즉시 반영. "만들기" 시 실제 node/line/polyline 객체로 커밋(현행 로직 재사용).

## 5. 인스펙터 변경 (비우기 — 확정 5)

- **funcgraph 인스펙터**: 상세 설정 행(끝 라벨·선 종류·선 모양·그래프 요소·구간 화살표)을 **제거**. **남기는 것 = 선 색·굵기(공용 "선" 섹션)뿐**(확정). 그 외는 모달에서.
- **coordplane 인스펙터**: **"그래프 편집…" 버튼 + 내보내기**만(현행 슬림 상태 유지). 상세 설정 없음.
- 재편집 = 그래프(평면) **더블클릭 → 통합 모달 다시 열기**(편집 모드).

> ⚠️ 가정 C: "곡선으로 변환"(수식 구동 끊고 일반 곡선화) 같은 escape hatch는 인스펙터에 남긴다(모달과 무관한 파괴적 변환). — 빼도 되면 알려주세요.

---

## 6. F 단축키 / 입구 통합 (확정 6)

- **F 누르면 = 이 통합 모달을 좌표 탭 포커스로** 연다(확정). 새 그래프를 처음부터 만들 때 좌표부터 확인하는 흐름.
- 선택된 좌표평면이 있으면 그 평면 위(편집 모드 유사), 없으면 새 좌표로 시작.
- 좌측 툴바 "그래프" 버튼 = 같은 모달(좌표 탭 포커스). 입구는 그래프 버튼 + F 둘 다 → **하나의 모달**, **둘 다 좌표 탭 먼저**.

---

## 7. 시각 수정 (회색 눈금 — 확정 1)

- `render/coordplane.js`의 **눈금 표시선(tick marks)** 색을 `color`(검정) → **회색**으로. 격자 회색(`GRID_LEVEL`)과 같은 톤 또는 그보다 약간 진한 회색.
- 축선·화살표·눈금 숫자·축 이름·원점 = **검정 유지**. 배경 격자 = 현행 유지.

> ⚠️ 가정 F: 눈금 표시선 회색 = 격자보다 약간 진한 중간회색(예: 명도 110~130). 격자와 완전히 같은 톤이면 눈금이 안 보일 수 있어 살짝 진하게. — 정확한 톤은 화면 보고 1회 조정.

---

## 8. 데이터 모델 영향 (최소 — 새 타입 없음)

- **새 객체 타입 안 만든다.** 기존 `coordplane`(축틀) + `funcgraph`(계열) + 표준 `node`/`line`/`polyline`(그래프 요소) 그대로.
- 이번 개편은 **UI 재배치 + 입구 통합 + 눈금 색**이 핵심. 저장 스키마 변화 없음(그룹 묶기·curveStyle·arrowVariant는 이미 반영됨, v0.54.32).
- 모달이 세팅하는 필드는 현행과 동일. 인스펙터에서 옮겨오는 것도 같은 필드를 모달에서 쓸 뿐.

## 9. Phase 계획

- **Phase 1 (핵심)**: ① 눈금 색 회색  ② 모달 2탭 분리(좌표/함수, 미리보기 고정)  ③ 함수 탭에 계열 종류·선 모양 통합.
- **Phase 2 (완결)**: ④ 함수 탭에 그래프 요소(표시점/수선/구간화살표 방향4옵션) 미리 세팅 + 미리보기 실시간  ⑤ 인스펙터 비우기  ⑥ F 단축키 통합 모달 연결.
- **Phase 3 (다듬기)**: 눈금 톤 미세조정, 편집 모드 탭 기억, 도우미 버튼·핸들 등 함수 도구 편의 이관 점검.
