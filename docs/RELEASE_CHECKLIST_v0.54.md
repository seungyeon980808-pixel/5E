## E. main 승격 절차 (순서 중요)

- [ ] ★ E-1. main도 이미지·manifest를 커밋해야 함(로컬 없이 동작 방침) → **main에서 `.gitignore`의 `images/*`·`manifest.json` 제외 규칙 삭제** 후 이미지+manifest 커밋. `_backup_old_images/`·`*.bak` 제외는 유지
- [ ] ★ E-2. 병합: `main`은 hub보다 0커밋 앞이므로 `git checkout main && git merge integration-hub` = fast-forward, 충돌 없어야 정상. **충돌이 나면 중단하고 원인 확인** (누가 main에 직접 커밋했다는 뜻)
- [ ] ★ E-3. main은 worktree(`5E_main` 폴더)에 체크아웃돼 있음 — hub 폴더에서 `git checkout main` 하면 "already checked out" 에러. **`5E_main` 폴더에서** 병합할 것
- [ ] ★ E-4. push 전 용량: 이미지 295MB — GitHub 단일 push 2GB 한도 내라 1회 push 가능. 단 시간이 걸리니 타임아웃처럼 보여도 기다릴 것
- [ ] ☆ E-5. push 후 원격 확인: `git log origin/main -1`이 로컬과 일치하는지
